🟢 Fase 2 — Inteligencia y Datos

RAG, persistencia, caché y sesiones individuales

RAG Pipeline Reemplaza knowledge_base.py. Documentos indexados como embeddings en Azure AI Search.
+
Base de Datos Cosmos DB / PostgreSQL. Persiste conversaciones, sesiones y métricas.
+
Redis Cache Cachea respuestas frecuentes. Reduce ~30-40% llamadas a OpenAI.
+
Sesiones Una instancia de chat por usuario (JWT/cookie). Elimina estado global compartido.

🟠 Fase 3 — Fine-Tuning y Modelo Propio

Modelo personalizado, clasificación de intenciones y evaluación automatizada

Intent Classifier Clasifica la intención antes del LLM: FAQ → caché, Producto → RAG, Queja → humano.
+
Modelo Fine-Tuned GPT-4o-mini entrenado con datos reales de BCS. Sin system prompt extenso.
+
LLM-as-Judge Evalúa respuestas: precisión, tono, alucinaciones. Pipeline automático de calidad.
